# Productos GREENPOW

## Narrativa de producto

GREENPOW se organiza en cinco areas de producto que posicionan a la empresa como tecnologia de infraestructura cloud orientada a producto.

## 1. Optimizacion de cloud publica

Enrutamiento de cargas sensible al carbono, optimizacion de costes cloud y visibilidad de Scope 2 para organizaciones que usan proveedores de cloud publica.

### Valor principal

Ayudar a los equipos a decidir donde y cuando debe ejecutarse la computacion comparando intensidad de carbono, precio de energia, disponibilidad de infraestructura, coste y rendimiento.

### Casos de uso

- Planificacion de cargas de trabajo.
- Seleccion de regiones y zonas.
- Comparacion de coste y carbono en cloud.
- Insumos para reporting de Scope 2.
- IA y procesamiento batch sensible al carbono.

## 2. VPS y hosting web

VPS sostenible y hosting web gestionado para empresas que migran desde entornos de hosting heredados.

### Valor principal

Ofrecer una entrada practica para organizaciones que necesitan infraestructura mas limpia sin gestionar operaciones cloud complejas.

### Casos de uso

- Hosting de sitios web empresariales.
- Migracion VPS.
- Hosting gestionado.
- Opciones de hosting con menor carbono.
- Modernizacion de infraestructura.

## 3. Cloud privada

Infraestructura soberana, hibrida y empresarial para organizaciones con requisitos de control, cumplimiento, residencia de datos o rendimiento.

### Valor principal

Ofrecer modelos operativos tipo cloud con mayor gobierno y control sobre donde se ejecuta la infraestructura.

### Casos de uso

- Cloud soberana.
- Infraestructura hibrida.
- Cargas reguladas.
- Modernizacion empresarial.
- Infraestructura dedicada para sistemas sensibles.

## 4. Infraestructura para agentes de IA

Capa cloud para organizaciones autonomas, flujos de trabajo habilitados por IA y operaciones agenticas.

### Valor principal

Proporcionar patrones de infraestructura para flujos de IA que necesitan computacion, contexto, planificacion y fiabilidad operativa.

### Casos de uso

- Hosting de flujos de agentes.
- Infraestructura de operaciones de IA.
- Cargas de IA programadas.
- Ubicacion de computacion IA sensible al carbono.
- Infraestructura para procesos empresariales autonomos.

## 5. Cortex

Cortex es una capa estatica de memoria organizativa para agentes autonomos.

### Valor principal

Convertir conocimiento institucional, memoria de procesos, contexto de decision y estructura operativa en una capa utilizable por agentes de IA y sistemas de soporte a la decision.

### Casos de uso

- Contexto organizativo para agentes de IA.
- Memoria de procesos.
- Estructura de soporte a la decision.
- Continuidad del conocimiento.
- Infraestructura para organizaciones autonomas.
