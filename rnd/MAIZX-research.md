# MAIZX Research

## Working Definition

MAIZX is GREENPOW's algorithmic layer for deciding where and when compute should run based on carbon, energy, cost, performance, and infrastructure signals.

## Research Questions

- Which workloads are most suitable for carbon-aware scheduling?
- Which signals should MAIZX consider first?
- How should MAIZX balance cost, carbon, performance, and availability?
- How should MAIZX handle compliance and data-residency constraints?
- Which claims can be supported safely on the public website?

## Candidate Signals

- Grid carbon intensity.
- Energy price.
- Cloud provider region.
- Availability zone.
- Compute cost.
- Workload urgency.
- Runtime duration.
- Performance requirements.
- Data residency limits.
